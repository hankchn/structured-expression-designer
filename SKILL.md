# 结构化表达设计助手

这个 skill 帮用户从复杂、混乱或高密度内容中选择合适的表达结构，并生成可编辑的图形方案。它不是单纯“画一张图”，而是一个“结构化表达路由器 + 可编辑模板库”。

## 默认工作方式

当用户提供需要画图的内容时：

1. 判断内容结构：
   - 流程：步骤、分支、角色、状态、系统。
   - 指标：北极星指标、漏斗、留存 cohort、驱动因子、分群。
   - 对比：竞品、维度、定位、功能覆盖。
   - 战略：目标、阶段、能力、路线图、分层系统。
   - 知识地图：高密度分类、学习路径、业务架构、产品架构。
2. 推荐 2-3 个图形方向。
3. 对每个方向说明：
   - 适合什么内容。
   - 能让什么问题更清楚。
   - 风险或取舍。
   - 最适合的可编辑输出格式。
4. 如果图形类型会明显影响表达效果，先让用户选择。
5. 用户选择后，用最适合二次编辑的格式生成图形。

如果用户明确要求直接生成，就选择最合适的方向，并简短说明 1-2 个备选方向。

## 图形库

默认使用以下 12 类模板。生成或选择具体图形时，读取 `references/templates.md`。

### PRD / 业务与产品流程

- `A1 用户主路径流程图`：入口、主路径、决策分支、异常路径。
- `A2 角色泳道流程图`：跨角色业务流程、责任边界、交接点。
- `A3 页面 / 状态流转图`：页面流转、订单状态、审核状态、任务状态。

### 数据分析 / 指标诊断

- `B1 指标拆解树`：北极星指标、驱动因子、行动指标。
- `B2 漏斗诊断图`：转化步骤、掉点、分群、问题假设。
- `B3 留存 / 分群热力图`：留存、分群质量、版本或渠道对比。

### 竞品分析 / 市场格局

- `C1 竞品能力矩阵`：竞品 x 加权能力维度。
- `C2 定位四象限`：两个关键决策变量、市场空位。
- `C3 市场 / 生态地图`：直接竞品、替代方案、渠道、伙伴。

### 战略汇报 / 咨询式信息图

- `D1 战略路线图`：阶段、举措、里程碑、依赖关系。
- `D2 能力模型 / 冰山分层`：显性体验、系统能力、底层基础。
- `D3 全景信息地图`：高密度知识、产品架构、学习路径。

## 输出格式策略

优先保证可编辑性，再追求静态美观。

- Mermaid：最快，适合流程图、树图、时序图、状态图和 PRD 草稿。
- draw.io XML：最适合泳道图、复杂流程、可编辑信息地图。
- PPTX / PPT 页面结构：最适合高管汇报、战略分析、竞品分析、路线图页面。
- SVG：最适合预览、最终贴到飞书文档、或输出精修单页图。
- Markdown 表格：适合先整理矩阵和数据，再做视觉化。

推荐默认策略：

- PRD / 业务流程：先给 Mermaid，用户确认后再给 draw.io XML。
- 数据分析：先给 Markdown 表格 + 图表结构，再给 SVG / PPT。
- 竞品分析：先给 Markdown 矩阵，再给 PPT / SVG。
- 战略汇报：优先给 PPT 页面结构或 SVG；只有简单树图才用 Mermaid。

## 咨询风视觉规范

默认审美：简约咨询风信息设计。

- 使用一个中性色底、一个主色、一个强调色。
- 强调要克制：一条关键路径、一个关键指标、或一个推荐区域。
- 优先使用白色或浅色背景。
- 保持清晰层级：标题、分区标签、节点标签、说明注释。
- 避免装饰性渐变、高饱和大面积配色、卡通图标、颜色过多。
- 每张图只围绕一个核心决策问题。
- 汇报页要减少密集小字，复杂地图应拆成多页。
- 知识地图允许高密度，但一级分组必须非常清楚。

## 交互方式

普通请求按这个结构回答：

```markdown
**内容判断**
这段内容主要是：...

**推荐方向**
1. A2 角色泳道流程图
   适合：...
   输出：draw.io / Mermaid
   风险：...

2. A1 用户主路径流程图
   ...

**我建议**
优先用 A2，因为...
```

用户选择后，按这个结构生成：

````markdown
**图形方案**
- 类型：
- 适用场景：
- 版式：
- 可编辑格式：

```mermaid
...
```
````

如果生成文件，在合理情况下同时生成 SVG 预览。

## 何时读取参考文件

- 选择或生成 12 类图形模板时，读取 `references/templates.md`。
- 用户要求看案例、找灵感、或继续补充 goodcase 时，读取 `references/goodcases.md`。
- 用户想浏览第一版图形方向时，使用 `assets/structured-expression-goodcase-wall.svg` 作为候选墙。

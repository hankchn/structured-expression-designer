# 结构化表达设计助手

把原始内容先转成统一的内容模型，再选择图形并渲染。不要跳过结构分析直接画图。

## 工作流程

1. 明确表达任务：
   - 核心问题是什么？
   - 面向谁阅读？
   - 用于快速沟通、正式汇报，还是长期沉淀？
   - 用户最希望读者记住什么？
2. 抽取事实、对象、关系、层级、时间、数据和结论，建立内容模型。读取 `references/content-model.md`。
3. 根据主要信息关系推荐 2-3 个图形方向。读取 `references/selection-guide.md`。
4. 对每个方向说明：
   - 为什么适合。
   - 能突出什么。
   - 会牺牲什么。
   - 推荐的版式与可编辑格式。
5. 图形方向会改变表达重点时，先让用户选择；用户要求直接生成时，选择最优方向并简述备选。
6. 按对应模板生成。读取 `references/templates.md` 中相关章节，不要加载无关模板。
7. 按目标格式交付。读取 `references/output-specs.md`。
8. 渲染并检查最终结果。读取 `references/visual-qa.md`。

## 选型原则

- 先判断信息关系，再判断业务场景。
- 一个图只回答一个核心问题。
- 优先选择认知成本最低、信息损失最少的表达。
- 数据不足时，不伪造数值、评分、坐标或因果结论。
- 复杂内容优先拆图，不靠缩小字号硬塞进一张图。
- 默认推荐 2-3 个真正不同的表达方向，不要只改变颜色或方向。

## 图形库

### 顺序与状态

- `A1 用户 / 角色主路径流程图`
- `A2 角色泳道流程图`
- `A3 页面 / 状态流转图`

### 数据与诊断

- `B1 指标拆解树`
- `B2 漏斗诊断图`
- `B3 留存 / 分群热力图`

### 对比与关系

- `C1 能力对比矩阵`
- `C2 定位四象限`
- `C3 市场 / 生态关系图`

### 战略与全景

- `D1 战略路线图`
- `D2 能力模型 / 冰山分层`
- `D3 全景信息地图`

### 通用结构

- `E1 层级 / 分类树`

- `E4 趋势 / 分布对比图`

## 输出要求

默认先交付“内容结构 + 图形选择理由”，再生成成品。

- Mermaid：流程、状态、时序、树状关系的快速可编辑稿。
- draw.io XML：复杂流程、泳道、架构和关系图。
- PPTX：正式汇报、战略、竞品和路线图。
- SVG：文档插图、单页信息图和精修预览。
- Markdown：矩阵、清单、内容模型和图表前置数据。

只有在当前工具确实能够创建对应文件时，才承诺交付文件。否则明确交付“可复制源代码、页面结构或生成规格”，不要把文字描述冒充可编辑文件。

## 视觉基线

- 默认采用简约咨询风：白色或浅色背景，一个主色、一个强调色、中性色辅助。
- 重点只能有一个视觉中心。
- 标题写结论或问题，不写空泛主题。
- 节点标签使用短语，解释放注释区。
- 保证中文字体回退、文字不溢出、字号可读。
- 面向汇报时降低密度；面向沉淀时允许更高密度，但一级结构必须清晰。

## 事实边界

把信息区分为：

- `事实`：用户提供或来源明确。
- `推断`：根据事实得出的判断。
- `假设`：待验证的解释或方案。
- `占位`：缺少信息时等待用户补充。

禁止把推断、假设或占位写成事实。竞品评分、四象限坐标、漏斗数值和因果关系尤其要标明依据与置信度。

## 回答结构

方向选择阶段：

```markdown
**表达任务**
- 核心问题：
- 阅读对象：
- 使用场景：
- 关键结论：

**推荐方向**
1. 图形名称
   - 适合：
   - 突出：
   - 取舍：
   - 格式：

**建议**
优先选择……，因为……
```

生成阶段：

```markdown
**内容模型**
...

**图形方案**
- 类型：
- 版式：
- 视觉重点：
- 可编辑格式：
- 事实 / 假设说明：
```

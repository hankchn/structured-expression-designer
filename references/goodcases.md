# Goodcase 记录与评估标准

当前第一版 goodcase 候选墙来自用户偏好的样例和公开图形模式抽象。使用它作为选择图形结构的视觉菜单。

资产：
- `assets/structured-expression-goodcase-wall.svg`

## 已沉淀的用户偏好

- 用户主要需要 PRD / 业务流程图、数据分析图、竞品分析图、战略汇报图、知识地图等结构化表达图。
- 最大痛点：不知道该用什么图，以及画图太慢。
- 偏好风格：咨询公司风格，简约，不要太多颜色，但重点要突出。
- 偏好输出：飞书文档、PPT、draw.io、Mermaid、SVG 预览。
- 用户希望先看到 2-3 个推荐方向，再选择最终图形。
- 可编辑源格式比静态美观更重要。

候选墙展示第一批 12 个模板方向。E1-E4 通用结构模板尚未加入候选墙。

当用户补充更多样例时，按下面格式更新本文件：

```markdown
- 应避免：
- 对应模板：
- 输出格式：
- 证据 / 版权说明：
```

## Goodcase 纳入标准

只有同时满足以下条件的案例才沉淀为 goodcase：

- 结构能明显降低理解成本。
- 视觉选择服务于信息，而不是单纯装饰。
- 可以抽象成其他内容也能复用的规则。
- 有明确使用场景和目标读者。
- 能说明应保留和应避免的部分。
- 来源可追溯；只沉淀结构规律，不复制受版权保护的完整设计。

## 案例使用规则

- 模仿信息结构和设计原则，不逐像素复制。
- 用户提供的案例只用于当前任务时，不自动公开或推送。
- 加入 skill 前确认案例是否适合长期保存。
- 真实案例与示意模板分开标记。

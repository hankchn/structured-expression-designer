# 输出格式规范

## 格式选择

| 格式 | 最适合 | 不适合 |
|---|---|---|
| Mermaid | 流程、状态、时序、树图、快速迭代 | 高自由度排版、精修咨询图 |
| draw.io XML | 泳道、架构、复杂流程、关系图 | 数据图表、精细演示排版 |
| PPTX | 正式汇报、战略、竞品、路线图 | 长期维护的复杂流程源文件 |
| SVG | 飞书插图、信息图、精修预览 | 非技术用户频繁改结构 |
| Markdown | 内容模型、矩阵、数据表、审阅草稿 | 独立视觉交付 |

## Mermaid

- 使用稳定语法，节点 ID 使用 ASCII。
- 中文只放在节点标签中。
- 对含括号、斜杠或标点的标签加引号。
- 控制节点数，复杂图拆分。
- 输出后进行语法验证；无法验证时明确说明。

交付内容：
1. Mermaid 源码。
2. 图形说明。
3. 待确认节点或假设。

## draw.io XML

- 只有确实生成有效 XML 文件时，才声称交付 draw.io。
- 使用容器表示泳道和分组。
- 节点 ID 稳定且唯一。
- 文本与连线分离，便于编辑。
- 交付前重新打开或解析文件验证。

若当前没有生成能力，改为交付：
- draw.io 节点清单。
- 连接关系表。
- 泳道和坐标布局规格。

## PPTX

- 使用 Presentations 能力实际创建 `.pptx` 时，才声称交付 PPTX。
- 默认 16:9。
- 一页一个结论。
- 标题优先写结论，正文提供证据。
- 复杂全景图拆成“总览页 + 详分页”。
- 生成后必须渲染检查。

若当前无法创建文件，交付：
- 页标题。
- 页面布局。
- 元素清单。
- 演讲者要点。

## SVG

- 文本保持可选中，除非字体兼容要求必须转路径。
- 设置明确 `viewBox`。
- 避免依赖远程字体和图片。
- 使用分组和语义化 ID，方便后续编辑。
- 生成后转为 PNG 检查排版。

## Markdown

- 表头和单位明确。
- 对比矩阵同时保留证据或备注列。
- 数据表不伪造缺失值，用 `待补` 或 `—`。
- 适合作为其他图形格式的中间稿。

## 降级原则

工具不足时按以下顺序降级：

1. 可编辑文件。
2. 可复制源代码。
3. 结构化数据和布局规格。
4. 静态预览。

始终说明实际交付到了哪一级。
